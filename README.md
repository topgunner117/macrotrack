# MacroTrack

A simple personal food + macro tracker. One self-contained `index.html` — no build step, no framework, all data stored locally in your browser.

## Features
- **Daily rings** for Calories / Protein / Carbs / Fat vs your targets (defaults: **1800 kcal · 130g P · 195g C · 55g F**).
- **Scan a nutrition label** with a photo → Gemini Flash-Lite reads it into structured numbers (you can fix any before saving).
- **Estimate a meal from photos** — Scan → **Meal** mode: snap 1–2 photos of a dish, say (🎤) or type what it is, and Gemini estimates the whole portion's calories + macros (editable before saving).
- **AI Coach** — the **Coach** tab answers food questions ("what should I eat to hit my last few g of carbs?") using today's log, your remaining macros, and your goal, via Gemini.
- **Edit any logged food** (incl. past days) — tap an entry to change servings/macros/name.
- **Search foods without a label** (chicken, rice, banana…) via the free USDA FoodData Central database (per 100 g).
- **Manual entry** for anything.
- **History** — every past day with an "On track / Off target" badge (on track = calories at/under target **and** protein at/over target).
- **Sync across devices** with a private sync key (optional — see below).
- Navigate days with the ‹ › arrows; tap the date to jump back to Today.

## Setup (one time)
1. Get a **free Gemini API key** at https://aistudio.google.com/apikey
2. Open the app → **Settings** → paste the key → **Save**.
3. (Optional) Get a free USDA key at https://fdc.nal.usda.gov/api-key-signup and paste it too. Without one it uses `DEMO_KEY`, which works but is rate-limited.

The Gemini free tier covers a personal amount of scanning at **$0**.

## Run it
**Easiest:** just double-click `index.html` — it opens in your browser and works.

**As a local server** (needed only if a browser blocks the camera on `file://`):
```
node server.js
```
then open http://localhost:4321

## Put it on your phone (installable PWA)
MacroTrack is a Progressive Web App — it installs to your home screen with its own icon and runs full-screen and offline (logging works offline; scanning/search need internet).

1. Open the deployed URL (Vercel) on your phone.
2. **iPhone:** Share → **Add to Home Screen**. **Android:** menu → **Install app** (or the install prompt).
3. Launch from the icon and enter your Gemini key once in Settings.

Icons are generated by `make-icons.js` (zero dependencies) — run `node make-icons.js` to regenerate them. The service worker is `sw.js`; bump its `CACHE` version when you change cached assets.

## Sync across devices
Your log is local-first (works offline in `localStorage`) and syncs to the cloud when a **sync key** is set, so the same data shows up on your phone and computer.

**Using it:** Settings → **Sync across devices** → **Generate key** → **Save & sync**. On another device, enter the *same* key (or open `https://YOUR-URL/?sync=YOURKEY`). Your food log and targets sync automatically (on load, on change, on focus, every ~45s). API keys stay per-device and are **not** synced. Keep the key private — anyone with it can read your log.

**How it works:** `api/sync.js` is a Vercel serverless function that stores one JSON blob per key in **Vercel Blob**. State is merged (union of entries by id; deletions carried as tombstones; targets last-write-wins), so logging on two devices never loses data.

**One-time backend setup on Vercel:**
1. In your Vercel project → **Storage** → **Create Database** → **Blob** → connect it to the project. This auto-adds the `BLOB_READ_WRITE_TOKEN` env var.
2. Redeploy (any push does it). That's it — `@vercel/blob` is already in `package.json`.
3. (Optional) Set a `SYNC_ALIASES` env var like `me:my-real-key` to use a short memorable alias without putting the real key in the repo.

Locally, `server.js` includes a file-backed mock of `/api/sync` so sync works in `node server.js` dev too (data in git-ignored `.sync-local.json`).

## Notes
- Food logs sync across devices when a sync key is set; otherwise they live only in this browser (localStorage). Your API keys always stay on the device.
- Model defaults to `gemini-2.5-flash-lite`; you can change it in Settings if Google renames it.
